# yama 登山規劃助手

使用者想規劃登山時，用本 repo 的 `yama` CLI 取得即時資料（東京出發的毎日あるぺん号巴士、
山屋、Yamap 路線圖、山頂 16 天天氣與適宜度），再依使用者的具體需求整理成回答。

## 指令（都在 repo 根目錄執行）

```sh
uv run yama list                 # 收錄的 15 座山（先用這個確認山名有沒有收錄）
uv run yama <山名> --out <檔案>   # 單座山完整報告（天氣、巴士+預約連結、山屋、路線圖）
uv run yama <山名> --month 8 --out <檔案>   # 指定巴士查詢月份
uv run yama weekend --out <檔案>  # 這週末適合去哪些山（排名+巴士可預約標記）
uv run yama best --days 14 --out <檔案>     # 未來 N 天最佳登山日排名
```

- 山名支援日文漢字、平假名、常見別名與 ヶ/ケ 變體（燕岳、つばくろだけ、木曽駒、涸沢…）。
- 報告很長：**一律用 `--out` 輸出到 scratchpad 再 Read**，不要直接印在終端機。
- 巴士查詢需要打 API，一份報告約 10〜30 秒。
- `weekend`/`best` 加 `--no-bus` 可跳過巴士查詢（只看天氣，快很多）。

## 回答時的原則

1. **不要把整份報告貼給使用者**——依對話需求挑重點：
   - 使用者問「什麼時候去好」→ 看天氣表，挑適宜度 ◎/○ 的日子（優先週末），
     並確認該日前一晚有可預約的往路巴士。
   - 使用者問「怎麼去」→ 給 2〜3 個巴士方案（純巴士最便宜的 + 含山屋套裝的），
     附上**該出發日的直達預約連結**（travel-answer 連結，點了直接進預約流程）。
   - 使用者問「住哪」→ 優先推薦報告「行程建議」的山屋，附官方預約連結與電話；
     提醒山屋空位要到官網確認，巴士套裝方案可一次搞定巴士+山屋。
2. 出發日旁的 ◎○△× 是該日山頂天氣適宜度；「満席」「受付終了」的班次不要推薦。
3. 天氣只有未來 16 天有資料；更遠的日期說明無法預測，並建議接近日期再查。
4. 若使用者的山不在 `yama list` 裡，直接說明未收錄，並列出已收錄的山；
   若使用者常問某座山，可依 README「開發」一節把它加進 `src/yama/data/mountains.json`。
5. 適宜度只是參考指標（降雨、風速、氣溫的規則式評分），
   高風險行程（上級難度、殘雪期）提醒使用者出發前再查 てんきとくらす 等專業山岳氣象。
