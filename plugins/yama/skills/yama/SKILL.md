# yama 登山規劃助手

使用者想規劃日本登山時，用 `yama` CLI 取得即時資料（東京出發的毎日あるぺん号夜行巴士、
山小屋、Yamap 路線圖、山頂 16 天天氣與適宜度），再依使用者的具體需求整理成回答。

## 執行方式

CLI 透過 uvx 直接從 GitHub 執行，**不需要 clone 或安裝**（需要有 [uv](https://docs.astral.sh/uv/)）：

```sh
uvx --from git+https://github.com/randy-tsukemen/hiking yama <指令>
```

首次執行會下載建置（約 10~30 秒），之後有快取。若使用者環境沒有 uv，
請先協助安裝：`curl -LsSf https://astral.sh/uv/install.sh | sh`。

## 指令

```sh
... yama list                    # 收錄的 15 座山（先確認山名有沒有收錄）
... yama <山名> --out <檔案>      # 單座山完整報告（天氣、巴士+預約連結、山屋、路線圖）
... yama <山名> --month 8 --out <檔案>   # 指定巴士查詢月份
... yama weekend --out <檔案>     # 這週末適合去哪些山（排名+巴士可預約標記）
... yama best --days 14 --out <檔案>     # 未來 N 天最佳登山日排名
... yama go <山名> [--when weekend|best|日期] [--party N]  # 一鍵成案：驗證天氣→巴士→房間，直接給唯一建議
... yama rooms <course_no> <日期>    # 查套裝方案的山屋「房間」空位（巴士有位≠房間有位）
```

- 山名支援日文漢字、平假名、常見別名與 ヶ/ケ 變體（燕岳、つばくろだけ、木曽駒、涸沢…）。
- 報告很長：**一律用 `--out` 輸出到暫存檔再 Read**，不要直接印在終端機。
- 巴士查詢需要打 API，一份報告約 10〜30 秒。
- `weekend`/`best` 加 `--no-bus` 可跳過巴士查詢（只看天氣，快很多）。

## 回答時的原則

0. 依使用者意圖選模式：
   - **提前規劃**（提到朋友、指定較遠月份、問「有哪些方案」）→ 完整報告
     `yama <山名> --month N`，列出巴士方案與各出發日預約連結——超過 16 天
     沒有天氣資訊沒關係，**先搶位比等天氣重要**（太晚訂就沒位了）。
   - **近期說走就走**（「這週末」「最近哪天好」）→ `yama go` 一鍵成案。
   不確定時先問一句「想先訂之後的日期，還是看近期天氣好的日子？」。
   提前規劃的回覆結構：**完整路線清單**（距離/爬升/標準時間/コース定数/日程，
   全部列出不要只挑一條）→ 每條路線標注適合的方案（日帰り路線→夜行日帰り往復、
   山頂泊→山小屋セット、A進B出縱走→注意復路上車點）→ 方案清單含預約連結。
   路線決定天數與進出點，才能決定訂哪個交通與住宿。
1. **不要把整份報告貼給使用者**——依對話需求挑重點：
   - 問「什麼時候去好」→ 看天氣表，挑適宜度 ◎/○ 的日子（優先週末），
     並確認該日前一晚有可預約的往路巴士。
   - 問「怎麼去」→ 給 2〜3 個巴士方案（純巴士最便宜的 + 含山屋套裝的），
     附上**該出發日的直達預約連結**（travel-answer 連結，點了直接進預約流程），
     並**同時附上方案詳細頁連結**（`https://bus.maitabi.jp/detail.html?course_no=N`，
     供確認行程說明與乘車地點；注意沒有 /bus/ 前綴）。
   - 問「住哪」→ 優先推薦報告「行程建議」的山屋，附官方預約連結與電話；
     提醒山屋空位要到官網確認，巴士套裝方案可一次搞定巴士+山屋。
2. 出發日旁的 ◎○△× 是登山日（往路夜行巴士＝出發翌日）的山頂天氣適宜度；
   「満席」「受付終了」的班次不要推薦。
- **套裝方案的房間空位**：巴士的「受付中/催行決定」不代表山屋房間有空位。
  推薦含山屋套裝給特定出發日時，用 `yama rooms <course_no> <YYYY-MM-DD>` 查逐晚房間狀態
  （○=有空位、數字=剩餘數、RQ=請求受理、WT=候補、×=已滿）。

3. 天氣只有未來 16 天有資料；更遠的日期說明無法預測，並建議接近日期再查。
4. 若使用者的山不在 `yama list` 裡，直接說明未收錄，並列出已收錄的山。
5. 適宜度只是規則式參考指標；高風險行程（上級難度、殘雪期）提醒使用者
   出發前再查 てんきとくらす 等專業山岳氣象。
6. 回答的推薦內容用純文字友善格式（使用者可能轉傳）：避免 Markdown 表格與內嵌連結，
   網址直接貼、用「・」條列。
7. 使用者用什麼語言問就用什麼語言回答；山名、山屋名保留日文原名。
